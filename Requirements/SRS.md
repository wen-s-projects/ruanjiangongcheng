软件需求规格说明书 (Software Requirements Specification - SRS)
1. 引言 (Introduction)
1.1 系统目的
本项目旨在开发一款跨平台（Web + Android）的健康饮食管理系统。通过 AI 技术实现食物识图、营养分析、热量计算及个性化方案推荐，帮助用户精准记录饮食并管理健康目标。
1.2 范围
系统包括：
前端： 基于 Web 的管理控制台和 Android 移动应用。
后端： 处理 AI 识别请求、存储用户数据、维护食物营养数据库。
AI 模块： 图像识别、自然语言处理（推荐方案）。
2. 总体描述 (Overall Description)
2.1 产品背景
现代用户对健康管理需求增加，但手动输入热量繁琐。本系统通过“识图记录”降低用户操作门槛，辅以 AI 建议提高用户粘性。
2.2 用户特点
普通用户： 关注体重管理、健身或营养平衡，习惯使用手机随时记录。
管理员： 审核用户上传的新食物数据，维护数据库准确性。
2.3 运行环境
Web 端： 现代浏览器（Chrome, Edge, Safari）。
移动端： Android 8.0 及以上版本。
服务端： 云服务器，需支持大规模图片存储及 AI 模型调用。
3. 系统特性 (System Features)
3.1 核心功能模块
AI 识图分析： 用户拍摄或上传食物照片，系统识别食物种类并估算热量与营养成分。
多端同步： Win 网页端与安卓 App 数据互通。
饮食历史记录： 以日历形式展示，支持按月翻阅，点击特定日期查看详细饮食明细。
AI 交互区域： 提供对话窗口，基于用户当前数据推荐饮食方案或解答营养问题。
工具集： 包含热量换算器、BMI 计算器。
食物数据库： 包含大部分常见食物数据，支持用户上传新食物条目。
4. 外部接口需求 (External Interface Requirements)
4.1 用户界面 (UI)
4.1.1 Web 界面逻辑
主页： 顶部为导航与头像。左侧为日历形式的历史记录，右侧为当前数据（热量/营养）与目标对比。下方为 AI 交互区及带附件功能的文本框。
登录/注册页： 居中表单，包含用户名、密码及验证码。
个人设置页： 侧边栏菜单，包含修改用户名、密码、头像、编辑当前数据、退出登录及隐私条款。
4.1.2 移动端界面逻辑
首页 (Index)： 模块化设计。顶部为历史查看，中间为今日缩略图和目标进度，底部为实时数据及底部导航栏（记录/主页）。
个人主页： 顶部背景+头像+简介，列表式功能项（数据编辑、目标设置、计算器、提醒、隐私条款）。
交互逻辑： 登录/注册页面需包含“同意条款”勾选框。
4.2 硬件/软件接口
摄像头接口： 移动端调用摄像头进行实物拍摄。
数据库： 使用关系型数据库（如 MySQL/PostgreSQL）存储用户数据，使用非关系型数据库（如 Redis）处理缓存。
4.3 通信接口
协议： 使用 HTTPS 协议保证数据传输安全。
API 格式： RESTful API，数据交互格式为 JSON。
5. 非功能性需求 (Non-functional Requirements)
5.1 性能
AI 响应时间： 食物识别请求应在 3 秒内返回结果。
页面加载： 网页及 App 首页加载时间不超过 2 秒。
5.2 安全性
权限控制： 用户仅能访问和修改自己的私有数据。
数据加密： 用户密码必须通过哈希加密存储（如 BCrypt）。
合规性： 严格遵守隐私政策，收集用户信息需经用户勾选同意。
5.3 可靠性
可用性指标： 系统全年可用性不低于 99.9%。
数据备份： 核心数据库每日进行冷备份。
6. 其他需求
法律合规： 用户注册时必须强制勾选隐私条款。
可扩展性： 系统架构应支持未来接入更多穿戴设备数据（如智能手表心率）。
